# Source Priority and Evidence Grading

Use this guide to classify source reliability and reduce interpretation drift.

## Priority Order

| Priority | Source class | Typical examples | Handling rule |
|---|---|---|---|
| P1 | Authoritative requirements | Statutes, regulations, formal bulletins, official orders | Treat as controlling where applicable |
| P2 | Supervisory guidance | Examination priorities, interpretive statements, FAQs | Treat as directional unless clearly binding |
| P3 | Industry standards and practice | SRO guidance, association frameworks, carrier standards | Use to inform controls and execution expectations |
| P4 | Internal policy | Company standards, approval playbooks, claim libraries | Enforce internally; reconcile with higher-order sources |

## Evidence Grade Model

| Grade | Meaning | Minimum documentation |
|---|---|---|
| A | Direct authoritative text with current effective status | Citation, date, jurisdiction, applicability tags |
| B | Strong interpretive guidance with clear source ownership | Citation and rationale note |
| C | Industry practice indicator requiring validation | Source note plus escalation recommendation |
| D | Anecdotal or weak signal | Do not operationalize without confirmation |

## Intake Metadata Fields

Capture per source:

- Source ID
- Publisher/owner
- Source class (P1-P4)
- Publication and effective dates
- Jurisdiction and product/channel applicability
- Evidence grade (A-D)
- Notes on ambiguity or conflict
