# Customer Journey Mapping Template

Framework for mapping segment-specific customer journeys. Each priority segment gets its own journey map because different segments experience fundamentally different buying and usage processes.

---

## Journey Map Structure

### 9-Stage Model

```
┌─────────┐   ┌──────────┐   ┌──────────┐   ┌────────────┐   ┌──────────┐
│ Trigger  │──→│ Problem   │──→│ Solution  │──→│ Evaluation │──→│ Decision │
│          │   │ Awareness │   │ Explore   │   │            │   │          │
└─────────┘   └──────────┘   └──────────┘   └────────────┘   └──────────┘
                                                                     │
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌─────────────────┐    │
│ Advocacy │←──│ Value     │←──│ Expansion│←──│  Onboarding     │←───┘
│ / Churn  │   │ Realized  │   │          │   │                 │
└──────────┘   └──────────┘   └──────────┘   └─────────────────┘
```

## Stage-by-Stage Template

### Stage 1: Trigger
The specific event or realization that initiates the buying journey.

| Dimension | Detail |
|-----------|--------|
| **Trigger events** | [List specific events that push this segment into market] |
| **Emotional state** | [Frustrated? Excited? Anxious? Curious?] |
| **Internal narrative** | [What they say to themselves: "We can't keep doing X..."] |
| **Detection signals** | [How you can detect this trigger externally — intent data, signals, etc.] |
| **Typical timeline** | [How long from trigger to active research?] |
| **Your optimal intervention** | [What to do when you detect the trigger] |

**Mapping questions:**
- What specific event made them start looking? (Not "they needed a solution" — the actual event)
- Who noticed the problem first? (End user? Manager? Executive?)
- How long did the pain exist before action was taken?

---

### Stage 2: Problem Awareness
How the segment frames and articulates the problem.

| Dimension | Detail |
|-----------|--------|
| **How they frame the problem** | [Their language, their framing — not your product category] |
| **Search terms used** | [Actual queries — pull from SEO data, sales conversations] |
| **Information sources consulted** | [Peers, Google, analysts, communities, etc.] |
| **Key questions asked** | [What they're trying to understand at this stage] |
| **Misconceptions** | [Common misunderstandings about the problem or solution space] |
| **Emotional state** | [Pain level, urgency, confidence] |
| **Your optimal intervention** | [Content type, channel, message] |

---

### Stage 3: Solution Exploration
Where and how they research potential solutions.

| Dimension | Detail |
|-----------|--------|
| **Research channels** | [Specific sites, communities, review platforms, analysts] |
| **Content consumed** | [Blog posts, comparison pages, case studies, videos, webinars?] |
| **Time spent researching** | [Days/weeks before shortlisting] |
| **Shortlist criteria** | [What determines which solutions make the cut?] |
| **Typical shortlist size** | [How many options do they seriously consider?] |
| **Influencers in this stage** | [Who shapes their thinking — peers, analysts, consultants?] |
| **Your optimal intervention** | [Content, channel, offer — what moves them to evaluate you] |

---

### Stage 4: Evaluation
How they compare and assess solutions.

| Dimension | Detail |
|-----------|--------|
| **Evaluation method** | [Demo? Trial? POC? RFP? Side-by-side comparison?] |
| **Decision criteria (ranked)** | [From persona — the top 3-5 factors] |
| **Evaluation participants** | [Who is involved? Roles and concerns?] |
| **Competitive comparison points** | [What do they compare on? Where do you win/lose?] |
| **Deal breakers** | [What immediately disqualifies a solution?] |
| **Evaluation timeline** | [How long from first demo to decision?] |
| **Common friction points** | [Where do evaluations stall or fail?] |
| **Your optimal intervention** | [How to support evaluation — assets, access, proof points] |

---

### Stage 5: Decision
What tips the final decision and who has authority.

| Dimension | Detail |
|-----------|--------|
| **Decision maker** | [Title/role that has final sign-off] |
| **Decision process** | [Committee vote? Single decision maker? Consensus?] |
| **Decisive factor** | [The single thing that tips it — from win/loss data] |
| **Final objections** | [Last-minute concerns that surface] |
| **Procurement process** | [Security review? Legal? Budget approval? Timeline?] |
| **Competitive dynamics** | [Who are you most often compared to at this stage?] |
| **Your optimal intervention** | [What closes — case study, ROI calc, exec alignment, risk reversal?] |

---

### Stage 6: Onboarding
What successful activation looks like for this segment.

| Dimension | Detail |
|-----------|--------|
| **Time-to-value target** | [How quickly must they see first value?] |
| **Activation milestones** | [Specific actions that indicate successful onboarding] |
| **Common blockers** | [What causes onboarding to stall?] |
| **Support needs** | [Self-serve? Hand-held? Dedicated CSM?] |
| **Key stakeholders in onboarding** | [Who needs to be involved for success?] |
| **Drop-off risk points** | [Where do they give up during onboarding?] |
| **Your optimal intervention** | [Onboarding playbook specifics for this segment] |

**Activation milestone examples:**
- SaaS: First workflow created, first integration connected, first report generated, team member invited
- E-commerce: First purchase, account created + preferences set, wishlist populated
- Marketplace: First listing created, first transaction completed, profile verified

---

### Stage 7: Value Realization
When and how they experience the core benefit.

| Dimension | Detail |
|-----------|--------|
| **"Aha moment"** | [Specific experience where value clicks] |
| **Time-to-value** | [Typical days/weeks from activation to value realization] |
| **Value metrics they track** | [What they measure to judge success] |
| **Engagement patterns indicating value** | [Usage behaviors correlated with retention] |
| **Risk signals** | [Absence of these behaviors = value not realized] |
| **Your optimal intervention** | [How to accelerate and reinforce value realization] |

---

### Stage 8: Expansion
What triggers consideration of more product/deeper engagement.

| Dimension | Detail |
|-----------|--------|
| **Expansion triggers** | [From persona — hitting limits, new use cases, team growth] |
| **Expansion path** | [More seats? Higher tier? Additional products? More usage?] |
| **Decision process for expansion** | [Same buyer? New buyer? Self-serve upgrade?] |
| **Barriers to expansion** | [Budget, awareness, competing priorities] |
| **Timing** | [Typical time from initial purchase to first expansion] |
| **Your optimal intervention** | [In-app prompts, CSM conversations, trigger-based outreach] |

---

### Stage 9: Advocacy / Churn
What drives referral vs. attrition.

| Dimension | Detail |
|-----------|--------|
| **Advocacy triggers** | [What makes them recommend you?] |
| **Advocacy channels** | [Where they share recommendations — G2, LinkedIn, direct referral?] |
| **Referral program effectiveness** | [Do incentives work with this segment?] |
| **Churn triggers** | [From persona — specific leading indicators] |
| **Churn reasons (ranked)** | [Why they leave, with frequency data] |
| **Win-back potential** | [Are churned customers recoverable? Under what conditions?] |
| **Your optimal intervention** | [Advocacy: program design. Churn: intervention playbook] |

## Journey Mapping Best Practices

1. **Map from the customer's perspective, not yours.** The journey starts when THEY feel a trigger, not when they enter your funnel.
2. **Use real customer language.** Pull quotes from interviews, support tickets, and reviews.
3. **Quantify wherever possible.** "Most customers research on G2" → "67% of this segment visits G2 before requesting a demo."
4. **Identify the 3 biggest friction points.** Focus improvement efforts on moments where the most customers drop off.
5. **Validate with frontline teams.** Sales, CS, and support see the journey daily — their input catches blind spots.
6. **Different segments = different journeys.** Never assume a single journey map covers all segments.
7. **Update regularly.** Journey maps decay. Review quarterly and refresh with new data.
