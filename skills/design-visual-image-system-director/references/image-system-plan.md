# Image System Plan Reference

Use this reference when planning imagery for a website, campaign, or multi-page surface. The goal is to prevent isolated generation by assigning every visual a role inside a shared system.

## Planning Sequence

1. **Extract the site story**
   - Primary audience
   - Main offer or product promise
   - Trust drivers
   - Emotional tone
   - Conversion or comprehension goal

2. **Set the impact standard**
   - Competitive tier the imagery must stand beside
   - Buyer expectation and decision value
   - Strategic value signal the set must create
   - Ownable visual idea or signature moment
   - Tropes that would make the set feel generic, low-value, or templated

3. **Map the content hierarchy**
   - What must be understood first
   - What needs proof
   - What needs explanation
   - What needs warmth or human trust
   - What can be expressed abstractly

4. **Assign visual modes**
   - Photography: trust, human presence, relationship, realism
   - Hybrid photo/data art: modern operating context, technology-supported judgment
   - Diagrammatic illustration: systems, process, movement, transformation logic
   - Product or interface surfaces: capability, workflow, proof of function
   - Texture or still-life detail: atmosphere, pacing, brand materiality
   - Iconographic signal marks: navigation, quick concept reinforcement, small repeated cues

5. **Set mode ratios**
   - Use photography sparingly when repeated people scenes would feel stock or generic
   - Use diagrams for abstract or strategic content that would become clichéd as photography
   - Use texture and still-life details to create breathing room between human or system-led scenes
   - Use signal marks for small repeated moments, not as substitutes for key storytelling images

6. **Define shared visual DNA**
   - Palette and color grade
   - Lighting temperature and contrast
   - Camera distance or diagram density
   - Materials and environmental cues
   - Motif grammar: lines, nodes, paths, grids, frames, shadows, crop devices
   - Image treatment: grain, matte finish, duotone, overlay strength, edge softness

7. **Define variation rules**
   - Each major page gets a different visual mode or composition archetype
   - Adjacent images should vary camera distance, subject count, density, or metaphor
   - Human images should not repeat the same office, pose, gesture, or meeting setup
   - Abstract images should not repeat the same network, ring, or path motif without a reason
   - Hero images need stronger hierarchy and more negative space than supporting images

## Image Inventory Fields

Use this matrix before writing prompts:

| Field | Question |
|-------|----------|
| Placement | Where will this image appear? |
| Content job | What must this image help the viewer understand? |
| Strategic value signal | Why does this image make the offer feel worth the buyer's attention and budget? |
| Emotional job | What should the viewer feel? |
| Visual mode | Which mode best serves the job? |
| Subject / metaphor | What exactly appears? |
| Composition | What crop, focal position, and depth are needed? |
| Shared DNA | Which system traits must carry through? |
| Variation rule | How is this different from the other images? |
| Must include | What is required? |
| Must avoid | What would make it generic, wrong, or redundant? |
| Acceptance criteria | What makes this image usable? |

## Common Failure Modes

- **Stock spiral:** every image becomes smiling people in similar workspaces
- **Overlay inflation:** data lines and holograms appear everywhere until they lose meaning
- **Mood-board drift:** each image looks good alone but belongs to a different brand
- **Polite low value:** images are tasteful and cohesive but do not raise buyer confidence or perceived value
- **Category default:** the set looks like any consulting, SaaS, coaching, or stock brand with the logo removed
- **Literalism:** abstract strategy pages become photos of people pointing at boards
- **Sameness disguised as cohesion:** same palette, same scene, same camera distance, same idea
- **Prompt amnesia:** later prompts say "match prior images" without repeating the actual shared rules
- **Crop neglect:** generated compositions fail once placed under nav, type, or responsive cropping

## Strong System Checks

A strong image system can answer these questions:

- Why does this image exist?
- What buyer value does this image signal?
- What would make this image feel expensive, strategic, and specific rather than merely professional?
- Why is this mode right for this content?
- What connects it to the rest of the suite?
- What makes it different from the next image?
- Where will the focal point land in the UI?
- What would make this image feel generic?
- What should never appear twice?
- Would a competitor be able to use this unchanged?
